Scene coverage

同一套语义 token，三种不同的工作语境。

admin 要求信息密度和操作秩序，docs 要求阅读节奏和导航清晰，weapp 要求触达安全与单手操作。这个 demo 关注它们的共同底色与边界，而不是做一套机械复用的万能模板。

默认策略：admin `light-first`，docs `light-first with dark bands`，weapp `light default with dark enhancement`。
Admin

左侧导航、顶部工具栏、能力带、石色摘要卡与规则化数据表面。整体更克制，允许局部 dark capability band 提升品牌感。

  • 保留 Ant Design / Element Plus 组件语义
  • 深色仅在关键模块和可切换主题中增强
  • 标题下调，不做 marketing 级巨型 hero
Docs

light-first 的编辑化文档入口，允许深绿 / 深蓝 hero 建立情绪，但正文与索引始终回归白色阅读画布。

  • 导航与阅读路径优先于装饰
  • coral 用于分类和协作标签
  • 入口卡强调薄边框和留白，而不是厚重阴影
Weapp

移动端通过深色头部带、浅色信息卡和稳定底部导航传递 Cohere 气质，不照搬 desktop split hero。

  • 首页、通知、快捷入口和个人中心优先
  • dark 模式保留浅色信息岛
  • 按钮、chip、底部导航必须满足触达安全
Token foundation

Light 与 dark 不是反色，而是两套层级节奏。

light 负责阅读和输入，dark 负责壳层、能力带与高对比氛围。primary action、editorial coral、research blue 三者角色分离，不互相侵占。

Phase 3 map

通过 demo 后，真实改造会遵循这些落点。

这不是“做完整站”的静态稿，而是 Phase 3 的真实实施地图。每一类页面都应该只拿到它需要的那部分 Cohere 语法。

React / Vue Admin

  • 先改 layout shell、登录页、workbench、列表页容器气质
  • 保持现有菜单、标签栏、搜索、设置抽屉的交互路径
  • dark 作为可切换主题增强，不把 CRUD 页面全部翻成深底

Docs

  • 优先改首页与入口页，不先动正文内容页
  • 把导航、阅读路径和工作区地图做成更清晰的产品入口
  • 深色能力带只出现在 hero、CTA 或治理模块

Weapp

  • 先改首页、个人中心、底部导航和关键摘要卡
  • 保留当前功能结构，不做移动端过度装饰
  • dark 模式优先验证头部带、卡片边界和导航可读性
