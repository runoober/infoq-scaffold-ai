# DESIGN

## 1. 设计目标

- 本设计服务的业务目标：以“用户导入”为例，展示一个典型后台功能在三端协作下的设计表达方式
- 对应 PRD：`doc/plan/2026-03-26-demo-user-import/PRD.md`

## 2. 触达端与范围

- Backend：需要提供导入数据与模板下载能力
- React：需要提供上传弹窗、模板下载、覆盖开关、结果反馈
- Vue：需要提供上传弹窗、模板下载、覆盖开关、结果反馈

## 3. 页面/界面结构

- 入口位置：用户管理页面工具栏中的“用户导入”入口
- 主体布局：中尺寸导入弹窗，包含拖拽上传区、覆盖开关、模板下载入口、确认/取消按钮
- 关键模块：文件上传、模板下载、覆盖选项、导入结果反馈
- 信息层级：先告诉用户上传什么文件，再告诉用户是否覆盖，最后告诉用户如何下载模板与查看导入结果

## 4. 交互流程

1. 管理员点击“用户导入”
2. 系统弹出导入对话框，展示上传区域和覆盖选项
3. 管理员可先下载模板，再选择文件
4. 点击确认后系统提交文件与 `updateSupport`
5. 成功时展示导入反馈；失败时展示明确错误原因

## 5. 状态设计

- Loading：上传中禁用重复提交，并显示上传中状态
- Empty：未选择文件时确认按钮不可用或给出明确提示
- Error：文件格式错误、解析失败、权限不足、服务端导入失败时必须给出明确提示
- Permission denied：无 `system:user:import` 权限时不展示入口或禁止操作
- Disabled/readonly：上传过程中禁止二次点击提交

## 6. 体验约束

- 关键可用性要求：管理员无需离开当前用户页即可完成模板下载、文件上传、覆盖选项选择
- 桌面端约束：弹窗宽度适中，避免导入说明被截断
- 移动端约束：若未来出现移动后台场景，需把拖拽区退化为点击上传，不强依赖拖拽
- 文案/图标/视觉约束：上传图标与提示文案应沿用当前前端组件库默认语义，不做多余装饰

## 7. 对实现的强约束

- 后端必须遵守：保持导入与模板下载接口职责清晰，错误信息不允许吞掉
- React 必须遵守：上传弹窗文案、覆盖选项、模板下载入口要与现有用户管理页面保持一致
- Vue 必须遵守：上传交互、覆盖选项、模板下载入口要与现有用户管理页面保持一致

## 8. 非目标与设计裁剪

- 本示例不要求重画全新页面
- 本示例不要求更换当前上传组件库
- 本示例只演示设计写法，不承诺本轮直接改动真实交互
