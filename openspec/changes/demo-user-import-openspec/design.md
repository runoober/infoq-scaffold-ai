# 设计：demo-user-import-openspec

## 设计目标

以“用户导入”为例，展示一个典型后台功能在后端、React、Vue 三端协作下的 OpenSpec 设计表达方式。

## 触达端与范围

- 后端：提供导入数据与模板下载能力
- React：提供上传弹窗、模板下载、覆盖开关、结果反馈
- Vue：提供上传弹窗、模板下载、覆盖开关、结果反馈

## 页面与交互结构

- 入口位置：用户管理页面工具栏中的“用户导入”入口
- 主体布局：中尺寸导入弹窗，包含拖拽上传区、覆盖开关、模板下载入口、确认/取消按钮
- 关键模块：文件上传、模板下载、覆盖选项、导入结果反馈
- 信息层级：先说明上传文件要求，再说明覆盖选项，最后提供模板下载和结果反馈

## 交互流程

1. 管理员点击“用户导入”
2. 系统弹出导入对话框，展示上传区域和覆盖选项
3. 管理员可先下载模板，再选择文件
4. 点击确认后系统提交文件与 `updateSupport`
5. 成功时展示导入反馈；失败时展示明确错误原因

## 状态设计

- 加载中：上传中禁用重复提交，并显示上传中状态
- 空状态：未选择文件时确认按钮不可用或给出明确提示
- 错误态：文件格式错误、解析失败、权限不足、服务端导入失败时必须给出明确提示
- 无权限：无 `system:user:import` 权限时不展示入口或禁止操作
- 禁用态：上传过程中禁止二次点击提交

## 体验约束

- 管理员无需离开当前用户页即可完成模板下载、文件上传、覆盖选项选择
- 桌面端弹窗宽度适中，避免导入说明被截断
- 若未来出现移动后台场景，拖拽区应可退化为点击上传，不强依赖拖拽
- 图标与提示文案沿用现有组件库默认语义，不做多余装饰

## 对实现的强约束

- 后端：保持导入与模板下载接口职责清晰，错误信息不允许吞掉
- React：上传弹窗文案、覆盖选项、模板下载入口要与现有用户管理页面保持一致
- Vue：上传交互、覆盖选项、模板下载入口要与现有用户管理页面保持一致

## 非目标与裁剪

- 本示例不要求重画全新页面
- 本示例不要求更换上传组件库
- 本示例只演示 OpenSpec 设计写法，不承诺本轮直接改动真实交互
